# 团队管理系统 Team Manager

基于 Flask + SQLite 的轻量级团队日常管理工具，支持签到考勤、任务跟踪、甘特图、报表导出等功能，无需安装任何依赖，开箱即用。

---

## 部署与启动

### 环境要求

- Python 3.7+
- 无需联网，所有依赖已内置于 `wheels/` 目录

### 启动方式

**Windows**

```bat
双击 start.bat
```

**Linux / macOS**

```bash
python app.py
```

服务默认监听 `http://0.0.0.0:8080`，启动后 1.5 秒自动打开浏览器访问 `http://127.0.0.1:8080`。

如需修改端口，通过环境变量指定：

```bash
PORT=9000 python app.py
```

### 首次启动

首次运行会自动创建 `data/teammanager.db` 数据库文件并初始化表结构，无需手动操作。

**默认账号：**

| 账号 | 密码 | 权限 |
|------|------|------|
| `admin` | `admin123` | 超级管理员 |
| `zhangsan` | `123456` | 普通成员（示例） |

> 首次使用后请尽快修改 admin 密码。

### 局域网访问

服务绑定到 `0.0.0.0`，同局域网内其他成员可通过部署机器的 IP 地址访问，例如 `http://10.21.230.243:8080`。

---

## 升级方式

1. 停止当前运行的服务
2. 替换 `app.py` 和 `static/index.html` 两个文件
3. **保留** `data/teammanager.db`（数据库文件，升级不丢失任何数据）
4. 重新启动服务

数据库结构变更通过 `init_db()` 中的 `ALTER TABLE` 自动完成，无需手动迁移。

## 功能介绍

### 角色与权限

系统共三种角色，登录后按角色展示对应模块：

| 角色 | 说明 |
|------|------|
| **超级管理员**（admin） | 可跨组查看所有成员数据，管理成员账号，配置系统参数 |
| **普通管理员** | 可查看并管理本组成员的签到、任务、报表 |
| **普通成员** | 只能查看自己相关的任务和签到记录 |

管理员可在左侧栏底部一键切换到"成员视角"，以普通成员身份预览页面。

### 自动签到

每天首次登录系统时自动完成签到（记录为"出勤"），无需手动操作。

- 若当日已记录请假/调休，登录后会弹窗提示，可选择保留或修改为其他状态
- 超级管理员账号（admin）不参与自动签到

### 总览（管理员）

页面分三个区域：

- **风险任务**（红色卡片）：显示标记了"有风险"且未完成的任务数量，点击跳转到任务管理
- **逾期问题单**（黄色卡片）：状态为进行中且计划结束日期已过的问题单
- **今日待办**：今天到期或计划开始的任务列表

### 签到管理（管理员）

- 展示当月各成员签到汇总，包含工作天数、出勤、缺勤、迟到、请假、远程、出勤率
- 今日签到列表显示状态徽章和打卡时间
- 迟到天数可点击展开详细迟到记录（日期 + 打卡时间）
- 签到记录中显示成员登录时的客户端 IP 地址
- 操作按钮：
  - **修改今日**：调整指定成员当日签到状态和备注
  - **全部记录**：弹出该成员本月签到明细
  - **今日全员考勤**：批量录入当日全员状态，适合日常管理
  - **⚙️ 迟到设置**：配置迟到判定时间阈值（默认 09:00）

### 任务管理

支持四种任务类型，通过页签切换：

| 类型 | 状态流转 |
|------|---------|
| **需求** | 待处理 → 进行中 → 联调测试 → 已交付 → 已取消 |
| **问题单** | 待处理 → 进行中 → 已解决 → 已关闭 → 已拒绝 |
| **现场支撑** | 进行中 → 已完成 → 已取消 |
| **其他事务** | 进行中 → 已完成 → 已取消 |

**创建/编辑任务：**

- 必填字段：标题、计划开始日期、计划结束日期（标注红色 `*`）
- 负责人下拉包含全部活跃成员和管理员（管理员名称后标注"(管)"）
- 支持"父任务"字段，子任务在列表中缩进显示
- 任务设置为完成状态时，若未填写实际结束日期，系统自动设为当日

**自动风险标记：**

任务超过计划结束日期，或距截止日期 7 天内且进度低于 80%，系统自动标记为"有风险"。

**进度日志：**

点击任务名称可添加进度日志，每条日志包含日期、进度百分比、内容描述，用于甘特图日期高亮和成员视图展示。

### 成员视图（管理员）

以卡片形式展示每位成员的任务状态和进展，包含：

- 进行中任务数、风险任务警告
- 今日进度日志摘要

页面顶部可输入日期进行筛选，查看某天各成员的具体进展。

### 甘特图（管理员）

按成员展示任务时间线，支持按月或全年视图。

- **分组快速选人**：成员勾选框上方显示分组按钮，点击某组自动勾选该组所有成员，再次点击取消
- **日期高亮**：点击"今天"/"昨天"快速选定日期，当天有进展记录的任务行显示黄色左边框，进度条上显示竖线
- **分组层级展示**：多个组时按组名小标题划分区块
- **任务条颜色**：蓝色=进行中，绿色=已完成，红色=有风险，任务名前带彩色标签表示类型
- 点击任意任务条可打开该任务的进度日志面板

### 月度日历（管理员）

以日历形式展示当月每天各成员的任务分布，顶部可选择年/月，右侧可勾选/取消特定成员进行筛选。点击日志格子可查看当日进展详情。

### 成员管理

- 展示所有活跃成员列表（账号、姓名、角色、所属组）
- **新建成员**：填写姓名、登录账号、初始密码、角色、所属组；勾选"管理员"可创建普通管理员账号
- **编辑成员**：可修改姓名、密码、角色、分组、是否管理员
- **停用**：软删除，成员无法登录但历史数据保留，可重新开启
- **永久删除**：彻底移除该成员及所有关联记录，不可恢复
- 超级管理员 admin 账号不能被停用或删除

### 报表中心

提供多维度数据分析，支持按时间范围、分组、成员筛选：

- **交付排名**：按交付数量对成员排名，直观展示贡献
- **需求 / 问题单 / 现场支撑 / 其他**：分别展示各类型任务的完成、进行中、风险、逾期情况，支持按成员筛选对比
- **签到报表**：展示各成员出勤率走势，包含出勤、迟到、缺勤、请假等指标
- **Excel 导出**：签到和任务数据均可导出为 Excel 文件

### 外观主题（管理员）

提供 8 种配色主题：

| 主题 | 说明 |
|------|------|
| `dark-blue` | 深蓝（默认） |
| `dark-purple` | 深紫 |
| `dark-green` | 深绿 |
| `dark-red` | 深红 |
| `carbon` | 碳黑 |
| `ocean` | 海洋 |
| `forest` | 森林 |
| `sunset` | 日落 |

支持自定义背景图，主题和背景图设置保存在浏览器 `localStorage` 中，主题设置仅管理员可见。

## 技术说明

- **后端**：Python 3 + Flask，单文件 `app.py`（~1073 行）
- **前端**：原生 JavaScript SPA，单文件 `static/index.html`（~2600 行），无构建步骤，无外部依赖
- **数据库**：SQLite（WAL 模式 + 外键约束），文件位于 `data/teammanager.db`
- **依赖**：Flask、openpyxl 等全部内置于 `wheels/` 目录，运行无需联网
- **密码**：客户端 SHA-256 加密后传输，服务端存储 SHA-256 哈希值
